## Hunyuan3D 纹理 GLB Blender 显示异常 - 最终修复
- demo_textured_full.glb 在 Blender 显示"纹理揉杂",demo_textured.glb 正常
- 根因: export_self_contained_glb() 用 struct 手搓 glTF 二进制,虽然数据正确(同几何/同UV/正确MR打包),但 bufferView 排布/字节对齐与 Blender 原生导出不同,导致 Blender glTF 导入器 PBR 渲染异常
- 误判方向: 一度怀疑 remesh 打碎网格,但 demo_textured.glb 用同一网格正常,排除
- 关键结论: 项目已装标准库 trimesh 4.4.7 (含 PBRMaterial + export_glb) 和 pygltflib 1.16.3,根本不必手搓二进制
- 最终修复: 重写 export_self_contained_glb() -> 用 trimesh 标准 GLB 导出器(scene.export(file_type=glb)),PBRMaterial(baseColorTexture + 打包MR),删除手搓 struct 代码和 Blender 优先分支
- 验证: 导出的 GLB 符合 glTF2.0(magic正确/2图2纹理1材质/PBR结构对/trimesh可回读),已重新生成 demo_textured_full.glb
- 教训: glTF/GLB 这类二进制格式,优先用标准库(trimesh/pygltflib)而非手动拼装,避免对齐/规范坑
